# Architecture Decision Records

Implementation-specific technology decisions live here rather than in the PRD, per [`Adventure_Discovery_PRD_v1.1.md` → Engineering Standards](../Adventure_Discovery_PRD_v1.1.md#architecture-decision-records-adrs). Each ADR is forced by a specific roadmap milestone — see [`ROADMAP.md` → ADR tracker](../ROADMAP.md#adr-tracker) for status.

An ADR is "Not started" until its forcing milestone is actually reached — don't pre-decide these speculatively. Once accepted, a decision is revised through a new ADR that supersedes the old one, not by editing history.
